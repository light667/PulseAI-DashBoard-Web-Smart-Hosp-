Guide d'Installation - Première Configuration

Erreur Détectée

L'erreur Failed to fetch signifie que les tables n'existent pas encore dans Supabase. Suivez les étapes ci-dessous pour résoudre le problème.

1 Ouvrir Supabase Dashboard

Allez sur votre projet Supabase :

Ouvrir Supabase Dashboard

2 Accéder au SQL Editor

Dans le menu latéral gauche de Supabase, cliquez sur :

SQL Editor

Puis cliquez sur le bouton "New query" en haut à droite.

3 Copier le Script SQL

Ouvrez le fichier sql/complete_setup.sql dans votre éditeur de code et copiez TOUT son contenu.

Le fichier fait environ 290 lignes et commence par :

-- ==============================================================================
-- PULSEAI - SCRIPT SQL COMPLET
-- ==============================================================================

4 Coller et Exécuter

  1. Collez le contenu dans l'éditeur SQL de Supabase
  2. Cliquez sur le bouton "RUN" (ou appuyez sur Ctrl+Enter)
  3. Attendez la fin de l'exécution (environ 5-10 secondes)
✅ Succès : Vous devriez voir "Success. No rows returned" - c'est normal !

5 Vérifier l'Installation

Dans le SQL Editor, exécutez cette requête de vérification :

SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;

Vous devriez voir ces 5 tables :

  • hospital_services
  • hospitals
  • profiles
  • ratings
  • services

6 Tester l'Application

Maintenant que les tables sont créées, testez l'application :

Page de Test S'inscrire Dashboard
Dépannage
Si l'erreur persiste après avoir exécuté le script :
  • Erreur "relation does not exist" : Le script n'a pas été exécuté ou a échoué
  • Erreur "permission denied" : Vérifiez que les politiques RLS sont activées
  • Erreur "Failed to fetch" :
    • Vérifiez que l'API REST est activée dans Settings → API
    • Vérifiez que votre clé anon est correcte dans src/config.js
Pour tout réinitialiser :

Si vous voulez repartir de zéro, exécutez d'abord sql/reset_database.sql, puis sql/complete_setup.sql

Une fois l'installation terminée, vous pourrez gérer votre hôpital via le dashboard !
